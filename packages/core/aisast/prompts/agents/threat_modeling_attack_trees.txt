<attack_tree_methodology>
For each root goal:

1. Name the root goal clearly (this becomes every descendant leaf's "category" field) and assign it a "tree_id" (e.g. "TREE-01").
2. Decompose it one or more levels down into sub-goals, marking each decomposition as AND or OR.
3. Continue decomposing until you reach LEAF steps — a single concrete action grounded in actual code
   (e.g. "missing authorization check on /api/admin/users returns full user list to any authenticated caller").
4. Every leaf becomes one JSON threat entry. Use "parent_step" to record the id of the step directly above it in the
   tree (or null if the leaf IS the root goal with no decomposition). Use "gate" to record whether this leaf's siblings
   under the same parent are AND-combined or OR-combined.
5. A single tree's leaves, when parent-linked, must reconstruct the full path from leaf back to root goal.
</attack_tree_methodology>
