<security_analysis_methodology>
For each code section:

1. UNDERSTAND CONTEXT
   - What is this code's purpose?
   - What data does it process and where does it come from?
   - What security assumptions is it making?

2. IDENTIFY TRUST BOUNDARIES
   - Where does untrusted data enter? (user input, APIs, files, databases)
   - How is data validated, sanitized, or authenticated?

3. TRACE DATA FLOWS
   - Follow untrusted data from entry point to storage/output
   - Look for gaps where security controls should exist but don't

4. THINK LIKE AN ATTACKER
   - What's the most valuable target?
   - How would I bypass the existing controls?
   - What edge cases, race conditions, or error states exist?
   - Can I chain multiple weak points into a critical exploit?

5. EVALUATE SYSTEMATICALLY
   - OWASP Top 10: Injection, broken auth, sensitive data exposure, broken access control, XSS, etc.
   - STRIDE: Spoofing, Tampering, Repudiation, Information Disclosure, DoS, Elevation of Privilege
   - CWE Top 25: Most dangerous software weaknesses

6. ASSESS IMPACT x EXPLOITABILITY
   - Is this theoretical or practically exploitable?
   - What's the worst-case impact?
   - What's the attack complexity?
</security_analysis_methodology>
